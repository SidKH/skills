# Obstruction

- Link: https://www.deceptive.design/types/obstruction

## Definition

Place extra barriers, friction, or confusion in the way of actions that do not align with provider goals, especially when the preferred path is much easier.

## Audit Cues

- reject, opt-out, or privacy-protective actions requiring more steps than acceptance
- asymmetric button prominence or branching complexity
- unclear navigation for the disfavored path
- repeated confirmations, hidden toggles, or extra forms for declining
- interface flows that make it hard to tell whether the user successfully chose the less profitable option

## Generic Example

A privacy prompt allows acceptance in one click, but rejection requires opening a secondary panel and changing multiple settings with ambiguous labels.
